## Project Overview

CMS web application for a Taiwanese language organization. Public-facing site with news, events, about/contact pages, plus a protected admin panel for content management.

## Commands

- `npm run dev` — Start dev server (localhost:3000)
- `npm run build` — Production build
- `npm run lint` — Run ESLint
- `npm test` — Run Vitest unit tests
- `npm run test:watch` — Run Vitest in watch mode
- `npm run test:e2e` — Run Playwright E2E tests (requires dev server + Supabase Local)
- `npm run test:e2e:ui` — Run Playwright E2E tests with UI
- `npm run test:all` — Run all tests (unit + E2E)

## Tech Stack

- **Next.js 16** (App Router, React 19, TypeScript)
- **Supabase** — PostgreSQL database, auth (email/password), file storage
- **shadcn/ui** (New York style, Radix UI + Tailwind CSS v4)
- **Tiptap** — Rich text editor (with image and link extensions)
- **React Hook Form + Zod** — Form handling and validation
- **Sonner** — Toast notifications

## Architecture

### Route Groups

- `src/app/(public)/` — Public pages (home, news, events, about, contact). Layout includes Navigation and Footer components.
- `src/app/admin/login/` — Unprotected login page with server actions in `actions.ts`.
- `src/app/admin/(dashboard)/` — Protected admin panel (news, events, pages, media CRUD). Layout includes Sidebar and Header.

### Auth & Middleware

- `src/middleware.ts` delegates to `src/lib/supabase/middleware.ts`
- Protects all `/admin/*` routes except `/admin/login`
- Unauthenticated users redirected to `/admin/login`; authenticated users on login page redirected to `/admin`

### Supabase Clients

- `src/lib/supabase/server.ts` — Server-side client (for Server Components and Server Actions)
- `src/lib/supabase/client.ts` — Browser client (for Client Components)

### Data Mutations

All CRUD operations use **Next.js Server Actions** (`actions.ts` files co-located with admin routes). Public pages use server-side Supabase queries with 60-second revalidation.

### Database Schema (in `supabase/migrations/`)

Four tables: `news`, `events`, `pages`, `site_settings`. RLS policies: public reads published content, authenticated users get full CRUD. Media stored in a public Supabase Storage bucket.

### Key Conventions

- Path alias: `@/*` maps to `src/*`
- UI components live in `src/components/ui/` (generated by shadcn CLI)
- Custom components in `src/components/` and `src/components/admin/`
- Types in `src/types/database.ts`
- Tailwind v4 with OKLCH color variables defined in `src/app/globals.css`
- Tiptap editor requires `immediatelyRender: false` for SSR compatibility

## Testing

- **Vitest** — Unit/integration tests in `tests/unit/`. Mocks Supabase client for Server Actions testing.
- **Playwright** — E2E browser tests in `tests/e2e/`. Three projects: `setup` (auth), `chromium` (admin tests with saved auth), `chromium-no-auth` (public/auth tests).
- E2E tests require Supabase Local (`supabase start`) and a test admin user. Set `TEST_ADMIN_EMAIL` and `TEST_ADMIN_PASSWORD` env vars or use defaults.
- Shared test fixtures in `tests/e2e/fixtures/` (db seeding, auth helpers).

## Environment Variables

Required in `.env.local`:
```
NEXT_PUBLIC_SUPABASE_URL=<supabase-project-url>
NEXT_PUBLIC_SUPABASE_ANON_KEY=<supabase-anon-key>
```
